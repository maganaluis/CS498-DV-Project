  • About the Visualization

    This visualization is an interactive slideshow which helps the user explore the Consumer Complaints Database. The dataset holds more than a million records from 2014-2018 but I've only included records from 2016-2018, since there wasn't anything interesting back then. The dataset required a lot of data wrangling and scrubbing, for which I've used Pandas, the application actually generates data dynamically as you go trough the slides, so the first thing you have to do to get started is to select a time period you would like to explore on the first visualization.

    Templates Scene

    Since I had to somehow make the Tableu visualizations on a hybrid structure, I opted for using the slideshow structure and start the user with the multiline-chart which allows him/her to select a time period, this time period is taken and the application generates two subsequent charts; a Stacked-Bar Chart and a pair of Donut-Charts at the end. The user is expected to scroll, or click on the left-hand side menu to move to the next visualization. They can come back to the first slide and make a different selection, which generates different data on the other charts.

    Annotations

    All charts are titled properly to indicate what they are trying to show, outter text anotations are minimal, I've only added them on the first graph to help the user understand that the next graphs will be generated dynamically. All charts have tooltips, proper legends, and they also display current the time-period for which they are showing data. On the Stackedbar-graph you can see it on the Y axis, and on the Donut-charts you can see it on the legend, this anotation changes dynamically.

    Parameters and Triggers

    The main trigger in the visualizations is when you make a time-period selection on the first chart, when this happens two API calls are made to filter data, wrangle it, and generate two new charts. This process happens in the background so there is no transition which the user can observe but in the background these are actually parameters being passed to the subsequent charts. The Donut-Charts at the end have subtle triggers that allow the user identify parts of the chart once they are clicked on.
